# 「亲情健步」产品功能清单

## 一、产品概述

面向家庭（尤其是老年人+子女）的微信小程序运动打卡应用。通过步数打卡→赚家庭币→兑换奖品/许心愿的闭环，激励老人坚持运动，增进家庭互动。

**技术栈**：微信小程序原生开发 + 微信云开发（云函数 + 云数据库 + 云存储）

---

## 三、功能详解

### 3.1 首页「家庭」

#### 未加入家庭（欢迎页）

- 显示欢迎引导，提供「创建家庭」和「加入家庭」两个入口按钮
- 点击后跳转到「我的」页面执行对应操作

#### 已加入家庭（主界面）

**家庭概览卡片**
- 显示家庭名称
- 横向展示所有成员：头像 + 昵称 + 今日步数
- 底部汇总今日家庭总步数

**金币横幅**（今日获得金币 > 0 时显示）
- 显示今日获得金币数和累计金币数

**今日排行榜**
- 默认显示前 3 名，超过 3 人时可展开/收起
- 排名标识：🥇🥈🥉 + 数字序号
- 每行：排名 + 头像 + 昵称 + 步数
- 第 1 名步数金色高亮，未打卡成员显示 0 步排在最后

**家庭动态广场（Feed 流）**
- 空状态引导文案
- 每条动态包含：
  - 头像 + 昵称 + 相对时间（"刚刚/X分钟前"）+ 类型标签
  - 类型标签：🚶打卡 / 🎁兑换 / 🌟心愿 / ✅实现 / 💪健身 / 💬分享
  - 打卡类型额外显示步数和获得金币数
  - 心情标签、文字内容
  - 图片区：单张全宽，多张三列网格，点击全屏预览
  - 互动区：点赞（支持取消）+ 评论
  - 评论列表：支持回复特定用户
- 分页加载，每页 20 条

**评论输入栏**
- 底部弹出，半透明遮罩
- 支持回复特定用户（placeholder 显示"回复 XXX:"）
- 支持键盘确认键直接发送

---

### 3.2 打卡页「今日打卡」

**步数展示卡片**
- 大号步数数字（千分位格式化）
- 进度条（以 8000 步为满格）
- 三档目标展示：3000 步 +1 币 / 5000 步 +2 币 / 8000 步 +3 币，达标项高亮
- 「同步微信运动步数」按钮
- 显示上次同步时间

**同步步数流程**
1. 调用 `wx.getWeRunData()` 获取加密数据
2. 通过云函数解密获取步数列表
3. 取今天的步数并缓存到本地
4. 步数 > 0 时提示可获得的金币数
5. 未授权时弹窗引导去设置页授权

**打卡记录区**
- 8 个心情标签可选：😊开心 / 🌤️天气好 / 💪有劲 / 🌸看到花 / 🐦听到鸟 / 🏃走得快 / 😌舒服 / 🌅风景美
- 文字输入框（最多 200 字）
- 添加照片（最多 3 张，支持相册/拍照，上传到云存储）
- 「完成打卡」/「更新打卡」按钮

**打卡流程**
1. 检查是否已加入家庭
2. 按步数计算金币
3. 写入/更新 `checkins` 集合
4. 更新用户 `coins`、`totalSteps`、`totalCheckins`
5. 发布打卡动态到 `feeds`
6. 刷新日历数据

**鼓励提示**（步数 < 3000 且 > 0 时）
- 显示"还差 X 步就能获得金币啦"引导文案

**打卡日历**
- 月份切换（不能超过当前月）
- 打卡日绿色圆点标记，今天橙色高亮
- 连续打卡天数徽章
- 月度统计：打卡天数 / 获得金币 / 总步数

---

### 3.3 兑换页「奖品兑换」

**金币余额卡片**
- 显示当前家庭币余额

**心愿卡区域**

*许心愿入口*
- 🌟 图标 + "许个心愿" + 花费提示
- 管理员额外显示「设置价格」按钮

*许心愿弹窗*
- 5 个预设模板：一起吃顿饭 / 打个视频电话 / 一起去散步 / 教我用手机 / 来家里坐坐
- 自定义输入框（最多 50 字）
- 显示花费金币数，金币不足时禁用

*心愿卡列表*
- 每张显示：状态图标（🌟待实现 / ✅已实现）+ 心愿内容 + 发起人昵称
- 待实现状态（非发起人可见）：
  - 「拍照实现」→ 选择/拍摄照片 → 上传 → 标记完成 → 发动态
  - 「直接标记完成」→ 确认弹窗 → 标记完成 → 发动态
- 待实现状态（发起人自己）：显示"等待家人实现..."
- 已实现状态：显示实现人昵称 + 可查看照片

*心愿卡价格设置弹窗*（管理员专属）
- 输入框设置每张心愿卡价格，保存到 `families.wishCost`

**奖品管理栏**（管理员专属）
- 「+ 添加」按钮跳转到添加奖品页

**可兑换奖品列表**
- 每个奖品卡片：
  - 类型图标：📸照片 / 🎤语音 / 🎬视频 / 💌文字
  - 奖品名称 + 描述
  - 价格（金色大号数字）
  - 剩余数量（"剩余 X / Y"）
  - 「兑换」按钮 / 「金币不足」灰色禁用
  - 管理员可预览内容、编辑、删除

**兑换流程**
1. 确认弹窗
2. 扣减用户金币 + 增加奖品已兑换数
3. 弹出成功面板，根据类型展示内容（照片/文字/语音播放/视频播放）
4. 发布兑换动态

**编辑奖品弹窗**（管理员专属，底部弹出）
- 可修改名称、描述、所需金币、类型、媒体内容、可兑换次数

### 3.4 添加奖品页

管理员从兑换页跳转进入。

| 表单项 | 说明 |
|--------|------|
| 奖品名称 | 批量添加时自动加序号 |
| 描述 | 可选 |
| 每个奖品所需金币 | 默认 30 |
| 奖品类型 | 📸照片 / 🎤语音 / 💌文字 / 🎬视频 |
| 照片模式 | 可多选最多 9 张，网格预览，支持预览/删除，批量创建（每张一个奖品） |
| 视频模式 | 可添加多个视频，缩略图预览，批量创建 |
| 语音模式 | 按住录音（最长 60 秒），可录多条，支持播放/删除，批量创建 |
| 文字模式 | 文本输入框，创建单个奖品 |
| 可兑换次数 | 默认 1 |

保存时逐个上传文件到云存储，逐个创建奖品记录。

### 3.5 我的页「个人中心」

**用户信息卡片**
- 头像：点击触发微信 `chooseAvatar` → 上传云存储 → 更新数据库
- 昵称：点击弹出编辑框修改
- 角色徽章：橙色"管理员" / 绿色"成员"
- 未设置昵称时显示引导提示

**统计数据卡片**（已加入家庭时）

| 统计项 | 说明 |
|--------|------|
| 打卡天数 | 累计打卡总天数 |
| 家庭币 | 当前余额 |
| 连续打卡 | 连续打卡天数 |
| 总步数 | 历史总步数 |

**家庭管理**

*已加入家庭时*
- 显示家庭名称，点击查看成员列表
- 邀请码（管理员可见），点击弹出邀请面板
- 家庭成员数，点击弹出成员列表弹窗

*未加入家庭时*
- 「创建家庭」→ 输入家庭名 → 创建 → 设为管理员 → 弹出个人信息设置引导 → 显示邀请码
- 「加入家庭」→ 输入 6 位邀请码 → 加入 → 设为成员 → 弹出个人信息设置引导

**设置**
- 切换角色：仅家庭创建者可见，管理员 ↔ 成员切换
- 大字模式：Switch 开关，全局放大字体（适老化设计）
- 关于：弹窗显示应用信息

**弹窗组件**
- 个人信息设置引导弹窗：创建/加入家庭后自动弹出，设置头像和昵称
- 家庭成员列表弹窗：头像 + 昵称 + 角色 + 打卡天数 + 金币数
- 邀请码面板：超大字号邀请码 + 复制按钮

## 四、云函数

| 云函数 | 触发方式 | 功能 |
|--------|---------|------|
| `login` | 前端调用 | 返回用户 openid/appid/unionid，用于身份识别 |
| `syncSteps` | 打卡页按钮 | 解密微信运动加密数据，返回步数列表。降级方案：CloudID 失败时尝试旧版 decryptData |
| `calcCoins` | 定时触发（每天 23:00） | 结算当天未结算打卡记录的金币，标记 `settled: true` 防重复；计算连续打卡天数，每连续 7 天额外奖励 3 币 |
| `manageFeed` | 前端按需调用（预留） | 支持删除动态（本人或管理员）、发送打卡提醒（订阅消息）、获取家庭统计 |

## 六、核心用户流程

### 6.1 新用户首次使用

```
打开小程序 → 云开发初始化 → 获取 openid
→ 首页显示欢迎页 → 点击"创建家庭"或"加入家庭"
→ 跳转到「我的」页面
→ [创建] 输入家庭名 → 创建家庭 → 设为 admin → 弹出个人信息设置引导 → 显示邀请码
→ [加入] 输入 6 位邀请码 → 加入家庭 → 设为 member → 弹出个人信息设置引导
→ 返回首页，显示家庭数据
```

### 6.2 每日打卡

```
打开「打卡」页 → 点击"同步微信运动步数"
→ 授权微信运动 → 云函数解密 → 显示今日步数
→ (可选) 选择心情标签 + 写文字 + 拍照
→ 点击"完成打卡" → 写入 checkins + 更新用户统计 + 发布动态
→ Toast "打卡成功！+X币" → 日历刷新
```

### 6.3 金币兑换奖品

```
打开「兑换」页 → 查看金币余额 → 浏览奖品列表
→ 点击"兑换" → 确认弹窗 → 扣减金币 + 增加已兑换数
→ 弹出成功面板（展示照片/播放语音/播放视频/显示文字）
→ 发布兑换动态
```

### 6.4 心愿卡闭环

```
[老人] 打开「兑换」页 → 点击"许个心愿" → 选模板或自定义
→ 确认弹窗 → 扣减金币 → 心愿卡发出 → 发布心愿动态
→ 心愿卡列表显示"🌟 等待实现"

[子女] 打开「兑换」页 → 看到待实现心愿
→ 方式A: "拍照实现" → 选择/拍摄照片 → 上传 → 标记完成 → 发布实现动态
→ 方式B: "直接标记完成" → 确认 → 标记完成 → 发布动态

[老人] 看到心愿卡变为"✅ 已实现" → 可查看实现照片
```

### 6.5 管理员设置奖品

```
打开「兑换」页 → 点击"+ 添加" → 跳转到添加奖品页
→ 填写名称/描述/金币数 → 选择类型
→ [照片] 选择最多 9 张 → 批量创建（每张一个奖品，自动编号）
→ [语音] 按住录音 → 可录多条 → 批量创建
→ [视频] 选择视频 → 可添加多个 → 批量创建
→ [文字] 输入文字 → 创建单个奖品
→ 逐个上传到云存储 → 逐个创建记录 → 返回兑换页
```

## 七、特色设计

### 适老化

- 大字模式开关，全局放大字体
- 预设心情标签，减少文字输入
- 心愿模板，降低使用门槛
- 邀请码排除易混淆字符（如 0/O、1/I/L）

### 激励机制

- 阶梯金币奖励：3000 步 +1 / 5000 步 +2 / 8000 步 +3
- 连续打卡每 7 天额外奖励 3 币
- 双向激励闭环：老人走路攒币兑换子女设的奖品 ↔ 老人许心愿让子女实现

### 数据安全

- 打卡幂等性：同日多次同步步数只更新不重复创建
- 奖品软删除：`active: false` 标记，保留历史数据
- 金币双重保障：客户端即时加币 + 云函数定时结算 + `settled` 标记防重复

## 八、已知待改进项

1. `redeemReward()` 和 `createWish()` 扣币操作缺少事务保护，存在并发风险
2. 订阅消息模板 ID 是占位符 `YOUR_TEMPLATE_ID`，需替换为真实 ID
3. 客户端加币 + 云函数定时加币存在重复计算风险
4. `db.js` 中 `getTodayRanking` 函数未被使用（冗余代码）
5. 邀请码无唯一性校验（碰撞概率极低但理论上可能重复）
