# 心愿卡功能 Spec

## 问题

当前奖品机制是一次性的：子女设奖品 → 老人兑换 → 兑完变灰 → 没有新动力。老人攒了家庭币却无处可花，缺乏持续运动的激励。

## 方案

新增**心愿卡**功能，与现有奖品共存。老人花家庭币兑换心愿卡，写下想要的心愿（如"一起吃顿饭"、"打个视频电话"），子女看到后去实现，完成后标记并附上照片。

## 需求

### 心愿卡基本规则
- 价格由管理员（子女）设定，默认10家庭币
- 管理员可随时修改心愿卡价格
- 老人可以选择预设模板，也可以自由填写心愿内容
- 心愿卡发出后显示在动态广场和商店页

### 预设心愿模板
1. 一起吃顿饭
2. 打个视频电话
3. 一起去散步
4. 教我用手机
5. 来家里坐坐
6. （自定义填写）

### 心愿卡流程
1. **老人发起**：在商店页点"许个心愿"→ 选模板或自由填写 → 花10币 → 心愿卡发出
2. **子女看到**：动态广场出现心愿卡动态，商店页显示待实现心愿列表
3. **子女实现**：子女点"已实现"→ 上传实现照片/视频 → 心愿卡标记完成
4. **老人查看**：老人可以看到完成的心愿卡和子女上传的照片/视频

### 心愿卡状态
- **待实现**：老人已发出，等待子女实现（黄色/橙色标识）
- **已实现**：子女标记完成并附上照片（绿色标识）

### UI 位置
- **商店页**：在奖品列表上方新增"心愿卡"区域
  - 老人（member）看到：许心愿按钮 + 自己的心愿历史
  - 子女（admin）看到：待实现的心愿列表 + 实现按钮
- **动态广场**：心愿卡发出和实现都作为动态显示

### 数据库
- 新增 `wishes` 集合：
  - `familyId`: 家庭ID
  - `openid`: 发起人（老人）
  - `nickName`: 发起人昵称
  - `content`: 心愿内容
  - `status`: pending / fulfilled
  - `fulfilledBy`: 实现人openid
  - `fulfilledPhoto`: 实现照片云文件ID
  - `fulfilledAt`: 实现时间
  - `createdAt`: 创建时间

## 验收标准

1. 老人在商店页能看到"许个心愿"按钮，点击后可选模板或自由填写
2. 花费10币后心愿卡发出，金币扣减正确
3. 金币不足时按钮显示"金币不足"且不可点击
4. 子女在商店页能看到待实现心愿列表
5. 子女点"已实现"后可上传照片，心愿状态变为已实现
6. 心愿卡发出和实现都在动态广场显示
7. 老人可以查看已实现心愿的照片

## 功能二：家庭排行榜

### 问题

家庭成员之间缺乏互动和比较，老人独自走路没有参照，容易失去动力。

### 方案

在首页添加家庭步数排行榜，展示家庭成员的步数排名，激发良性竞争。

### 需求

#### 排行榜内容
- 显示家庭所有成员的**今日步数**排名
- 按步数从高到低排列
- 显示：排名、头像、昵称、步数
- 第1名显示金色标识

#### 排行榜位置
- 首页（index）家庭动态区域上方，作为一个卡片展示
- 默认展示前3名，可展开查看全部

#### 数据来源
- 从 `checkins` 集合读取今日打卡数据
- 未打卡的成员显示0步，排在最后

### 验收标准

1. 首页能看到家庭步数排行榜卡片
2. 排名按今日步数从高到低排列
3. 显示头像、昵称、步数
4. 第1名有金色高亮
5. 未打卡成员显示0步

## 实现步骤

1. 数据库：创建 `wishes` 集合，在 `db.js` 中添加心愿卡相关操作函数
2. 数据库：在 `db.js` 中添加获取家庭成员今日步数排行的函数
3. 商店页 WXML/JS/WXSS：添加心愿卡区域（许心愿、心愿列表、实现心愿）
4. 首页 WXML/JS/WXSS：添加家庭排行榜卡片
5. 动态广场：心愿卡发出/实现时发送 feed 动态
6. 测试心愿卡流程：发起心愿 → 子女查看 → 标记实现 → 老人查看
7. 测试排行榜：多成员步数排名显示正确
