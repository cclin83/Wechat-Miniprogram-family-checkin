# 退出家庭功能

## 问题描述

用户误操作创建了新家庭后，无法退出当前家庭去加入其他家庭。需要新增「退出家庭」功能。

## 需求

### 退出规则
- 任何家庭成员（包括创建者）都可以退出家庭
- 退出后用户的金币、步数、打卡天数、连续打卡全部清零
- 退出后用户的 `familyId` 和 `role` 清空，回到未加入家庭状态
- 退出后从家庭的 `members` 数组中移除该用户的 openid
- 退出后用户的兑换记录（redemptions）清除

### 交互设计
- 在「我的」页面家庭管理区域新增「退出家庭」按钮
- 点击后弹出二次确认弹窗，提示"退出后金币和数据将清零，确定退出吗？"
- 确认后执行退出，提示"已退出家庭"
- 页面刷新回到未加入家庭状态

### 边界情况
- 如果创建者退出且家庭还有其他成员，家庭继续存在（不自动解散）
- 如果最后一个成员退出，家庭记录保留（不自动删除）

## 验收标准

1. 「我的」页面已加入家庭时显示「退出家庭」按钮
2. 点击后弹出确认弹窗，取消不执行任何操作
3. 确认退出后：用户 coins/totalSteps/totalCheckins/streak 归零，familyId/role 清空
4. 确认退出后：家庭 members 数组移除该用户
5. 确认退出后：用户的 redemptions 记录清除
6. 退出后页面显示「创建家庭」和「加入家庭」入口
7. 退出后可以正常加入其他家庭

## 实现步骤

1. `db.js` 新增 `leaveFamily(openid, familyId)` 函数
2. `profile.js` 新增 `leaveFamily()` 方法，含确认弹窗逻辑
3. `profile.wxml` 在家庭管理区域添加「退出家庭」按钮
4. 推送到 GitHub
